ABOUT ME自己紹介

ゲーム開発に情熱を持つエンジニアです。 Unity / C# によるマルチプレイヤーゲーム開発と、 C++ / DirectX 11 による独自レンダリングエンジン開発の両軸で 実践的な経験を積んでいます。

Mirror Networking を用いたサーバー権威型アーキテクチャを実践し、 ネットワーク上でのインベントリ同期・プレイヤー生成・オブジェクト管理など オンラインゲームの核となる技術を習得しています。また Unity ECS (DOTS) を活用した高パフォーマンスな設計にも取り組んでいます。

SQLデータベース を用いたゲームデータの永続化・管理も経験しており、 依存性注入 (Zenject)・ScriptableObject・インターフェース設計を組み合わせた データドリブンなゲーム開発を実践しています。

チームへの貢献を大切にし、読みやすいコード・明確なアーキテクチャ設計を 常に意識しながら開発を行っています。

🌾 Make-a-Farm — マルチプレイヤー農場ゲーム

Unity 6 + Mirror Networking で実装した、リアルタイムマルチプレイヤー農場ゲームのプロトタイプ。 サーバー権威型ネットワーク設計・依存性注入・インターフェース駆動設計を実践したフルスタック Unity プロジェクト。

Unity 6 C# Mirror Zenject Cinemachine URP New Input System
GitHub →

本プロジェクトは複数プレイヤーが同一ワールドで農業・取引・乗り物操作を楽しめる農場ゲームです。 ネットワーク越しにリアルタイムでゲーム状態を同期し、クライアント間の一貫性を保つ サーバー権威型アーキテクチャを採用しています。 農作物の成長タイマー・インベントリ・プレイヤー位置など全ての状態をサーバーが管理します。

✅ 実装した主な機能

🌐 マルチプレイヤーネットワーク

Mirror による Host/Client 方式。プレイヤーのスポーン・同期・切断処理を実装。

🌱 農業システム

種の設置→成長段階の進行→収穫までを完全にネットワーク同期。ゴーストプレビューで配置可否を視覚化。

🎒 ネットワーク同期インベントリ

SyncList<InventorySlot> による9スロットインベントリ。アイテムスタック・消費・UI連動を実装。

🏪 ショップシステム

Zenject の Factory パターンでショップカードを動的生成。インタラクション起動 UI を実装。

🚗 乗り物システム

WheelCollider を用いた車両物理。乗降・Cinemachine カメラ切換・ハンドル操作を実装。

🖱️ インタラクションシステム

レイキャスト + IInteractable インターフェースによるドア・農作物・ショップ・乗り物の統一操作。

📦 アイテムデータベース

ScriptableObject でアイテム定義。Zenject でシーンに注入し ID 検索可能なデータドリブン設計。

🚀 描画最適化 (Draw Call削減)

GPUインスタンシング・Static Batching・動的バッチングを適用。Draw Call を 286 → 88 に約 70% 削減し、FPS を 111 → 135 以上に改善。

🎮 プレイヤーコントローラ

カメラ相対移動・スプリント・ジャンプ・アニメーション連動を New Input System で実装。

✅ ゲームの主な機能

🎯 タレット操作システム

マウスで砲台を回転・照準。左クリックで弾を発射。ブロックに命中させてラインを消去。

🧱 テトリス式ブロック管理

ラインが揃うと消去してスコア加算。ブロックが危険ゾーンに到達するとゲームオーバー。

🎁 スペシャルブロック

「プレゼントブロック」を破壊するとボーナスアイテムを獲得。ゲームプレイに変化を加える。

⚡ パワーアップシステム

弾のダメージ・速度・特性を変えるアイテムを収集。リロール機能でアイテム再選択も可能。

✨ ビジュアルエフェクト

Cartoon FX による爆発・ライン消去エフェクト、タイトルアニメーション、3D 雲背景を実装。

📈 ステージ進行

ステージが進むにつれてブロックの落下速度・密度が増加。徐々に難易度が上がる設計。

👾 Slime Invasion 3D — 3Dシューティングゲーム

Unity を使わず C++ と DirectX 11 のみ でゼロから実装した3Dシューティングゲーム。 レンダリングパイプライン・物理・アニメーション・サウンドをすべて独自実装し、 グラフィックスプログラミングの基礎を習得したプロジェクト。

C++ DirectX 11 HLSL Assimp Visual Studio DirectXTex
GitHub →

スライムが大量に押し寄せる3Dシューティングゲームです。 ゲームエンジンを一切使わず、Direct3D / HLSL シェーダー / Assimp による3Dモデル読み込みなどを すべて自前で実装することで、ゲームエンジンの裏側にある技術を深く理解しました。 47コミット・約半年間の継続的な開発を通じて完成させたポートフォリオ作品です。

✅ 実装した主な機能

🖥️ Direct3D レンダリングパイプライン

頂点バッファ・インデックスバッファ・定数バッファを手動管理。スワップチェーンからフレーム描画まで自前実装。

💡 ライティングシステム

Phong反射モデルによるディフューズ・スペキュラ・ポイントライトを HLSL ピクセルシェーダーで実装。

🎭 スキンドアニメーション

Assimp のボーンデータを読み込み、頂点シェーダーで骨格アニメーションを計算・適用。

💥 コリジョン検出

AABB / 球体ベースの当たり判定を独自実装。弾丸・プレイヤー・敵スライム間の衝突を管理。

🌟 パーティクルシステム

ビルボードレンダリングを用いた弾着エフェクト・爆発パーティクルを実装。

👾 敵スポーンシステム

ウェーブ形式でスライムを生成・移動・追跡させるAIロジックを C++ で実装。

🔊 オーディオマネージャー

Windows Audio API を用いたSE・BGM の再生管理システムを実装。

📷 カメラシステム

プレイヤー追従カメラとデバッグカメラを切り替えられる自由視点カメラを実装。
